## 15.1 Nav2란

**Nav2**는 ROS 2의 표준 자율주행 스택이다. 지도와 현재 위치를 입력받아, 목표까지의 경로를
계획하고, 장애물을 피하며, 모터 명령(`/cmd_vel`)을 낸다.

```text
입력: /map(지도) + /scan(실시간 장애물) + /tf(위치)
   ▼
[전역 경로계획]  지도 전체에서 목표까지 큰 경로
[지역 경로계획]  당장의 장애물을 피하는 짧은 경로 + 속도
   ▼
출력: /cmd_vel
```

내부적으로 Nav2는 여러 **액션 서버**(예: `navigate_to_pose`)로 동작한다 — 5장에서 배운
액션이 실전에서 이렇게 쓰인다.

## 15.2 [따라하기] Nav2 실행

지도와 함께 Nav2를 띄운다.

```bash
export TURTLEBOT3_MODEL=burger
# 터미널 1 — 시뮬(또는 실로봇 bringup)
ros2 launch turtlebot3_gazebo turtlebot3_world.launch.py
# 터미널 2 — Nav2 + 저장한 지도
ros2 launch turtlebot3_navigation2 navigation2.launch.py \
  use_sim_time:=True map:=$HOME/maps/my_world.yaml
```

RViz가 지도와 함께 뜬다.

---

## 15.3 [따라하기] 초기 위치 잡기 → 목표 보내기

Nav2는 처음에 "내가 지도의 어디에 있는지" 모른다. 사람이 알려줘야 한다.

1. RViz 상단 **`2D Pose Estimate`** 클릭 → 지도에서 로봇의 실제 위치·방향을 드래그로 지정.
   라이다 점들이 지도 벽과 겹쳐지면 위치추정(AMCL)이 수렴한 것이다.
2. **`Nav2 Goal`** 클릭 → 목표 지점을 찍으면, 전역 경로(선)가 그려지고 로봇이 출발한다.
3. 도중에 장애물을 두면, 지역 계획이 우회 경로를 만든다.

> ⚠️ 초기 위치가 어긋나면 로봇이 엉뚱하게 간다. 라이다-지도 정합이 안 맞으면 `2D Pose
> Estimate`를 다시 잡는다. 가장 흔한 트러블의 원인이다.

## 15.4 [따라하기] 코드로 목표 보내기

RViz 클릭 대신, 액션 클라이언트로 목표를 보낸다. Nav2의 `navigate_to_pose` 액션을 쓴다.

```python
import rclpy
from rclpy.node import Node
from rclpy.action import ActionClient
from nav2_msgs.action import NavigateToPose
from geometry_msgs.msg import PoseStamped


class GoalSender(Node):
    def __init__(self):
        super().__init__("goal_sender")
        self.client_ = ActionClient(self, NavigateToPose, "navigate_to_pose")
        self.send(2.0, 1.0)

    def send(self, x, y):
        self.client_.wait_for_server()
        goal = NavigateToPose.Goal()
        goal.pose.header.frame_id = "map"
        goal.pose.pose.position.x = x
        goal.pose.pose.position.y = y
        goal.pose.pose.orientation.w = 1.0
        fut = self.client_.send_goal_async(goal, feedback_callback=self.fb)
        fut.add_done_callback(self.accepted)

    def fb(self, msg):
        d = msg.feedback.distance_remaining
        self.get_logger().info(f"남은 거리: {d:.2f}m")

    def accepted(self, future):
        gh = future.result()
        gh.get_result_async().add_done_callback(
            lambda f: self.get_logger().info("목표 도착"))


def main(args=None):
    rclpy.init(args=args)
    node = GoalSender()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()
```

5장 액션 클라이언트와 구조가 **똑같다**. 2권 전체가 하나로 이어지는 순간이다.

## 15.5 구성 직관 — 비용지도(costmap)

Nav2는 지도 위에 **비용지도(costmap)** 를 얹는다. 벽 근처는 비용이 높아(부딪히면 위험)
경로가 자연히 벽을 피해 가운데로 난다. 전역 비용지도는 정적 지도 기반, 지역 비용지도는
실시간 `/scan`으로 갱신돼 갑자기 나타난 장애물을 피한다.

## 15.9 [워크드 예제] 여러 지점 순찰

목표를 큐에 넣고 하나가 끝나면 다음을 보내는 순찰 노드. 결과 콜백에서 다음 목표를 던지는
구조가 핵심이다.

```python
class Patrol(Node):
    def __init__(self):
        super().__init__("patrol")
        self.client_ = ActionClient(self, NavigateToPose, "navigate_to_pose")
        self.waypoints_ = [(2.0, 1.0), (2.0, -1.0), (0.0, 0.0)]   # 순회 지점
        self.idx_ = 0
        self.client_.wait_for_server()
        self.go_next()

    def go_next(self):
        if self.idx_ >= len(self.waypoints_):
            self.get_logger().info("순찰 완료"); return
        x, y = self.waypoints_[self.idx_]
        goal = NavigateToPose.Goal()
        goal.pose.header.frame_id = "map"
        goal.pose.pose.position.x = x
        goal.pose.pose.position.y = y
        goal.pose.pose.orientation.w = 1.0
        self.client_.send_goal_async(goal).add_done_callback(self.accepted)

    def accepted(self, future):
        future.result().get_result_async().add_done_callback(self.done)

    def done(self, _):
        self.get_logger().info(f"지점 {self.idx_} 도착")
        self.idx_ += 1
        self.go_next()         # 다음 목표
```

목표 도착(`done`)마다 다음 웨이포인트를 보내 무한·반복 순찰로 확장할 수 있다.

## 15.10 Nav2 구성요소 한눈에

Nav2는 여러 서버의 묶음이다. 문제 원인을 좁힐 때 이 구조를 알면 빠르다.

| 구성요소 | 역할 |
|---|---|
| AMCL | 지도 위 자기 위치 추정(`2D Pose Estimate`로 초기화) |
| Planner Server | 전역 경로 계획 |
| Controller Server | 지역 경로·속도 명령 생성 |
| Behavior Server | 복구 행동(후진·회전 등 막혔을 때) |
| BT Navigator | 위들을 행동트리로 지휘(`navigate_to_pose` 액션 제공) |

"로봇이 가다 멈춰 회전만 한다"면 복구 행동이 작동 중일 수 있고, 대개 위치추정(AMCL)이나
TF가 원인이다.
