# [집필자 내부 노트] 장별 공통 템플릿

> 이 문서는 출간 본문이 아니라 집필자가 장별 리듬을 맞추기 위한 기준입니다.

## 장 앞부분

각 장은 제목 바로 아래에 다음 흐름을 둔다.

1. **학습 목표**: 독자가 장을 끝낸 뒤 말하거나 실행할 수 있어야 하는 것.
2. **이번 장의 산출물**: 실제로 만들어지는 코드, 설정, 지도, 시뮬레이션 상태, 실행 결과.
3. **공통 학습 흐름**: 개념 → 따라하기 → 코드 해설 → 실행 확인 → 버전/환경 체크 → 트러블슈팅 → 연습문제 → 마무리 점검.

본문에 넣는 블록 형식:

```markdown
> **이번 장의 산출물**
> - ...
>
> **공통 학습 흐름**: 개념 → 따라하기 → 코드 해설 → 실행 확인 → 버전/환경 체크 → 트러블슈팅 → 연습문제 → 마무리 점검
```

## 장 중후반

트러블슈팅 전에는 다음 섹션을 둔다. 장마다 제목은 같게 유지하고, 내용만 장에 맞춘다.

```markdown
## 코드 해설 · 실행 확인 · 버전 체크

- **코드 해설 포인트**: ...
- **실행 확인 포인트**: ...
- **버전/환경 체크**: ...
```

## 장 끝부분

각 장은 다음 순서로 마무리한다.

1. 트러블슈팅
2. 연습문제
3. 마무리 점검
4. 다음 장 예고
5. 참고 자료

## 분량 기준

- 모든 장을 같은 페이지 수로 맞추지 않는다.
- README의 목표 페이지 수를 기준으로 장별 밀도를 맞춘다.
- 2권 프로젝트 장은 실행 로그, 화면 확인, 실패 사례가 들어가므로 1권 개념 장보다 실습 확인 섹션이 더 두꺼워져도 된다.
